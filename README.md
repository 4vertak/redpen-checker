# RedPen — домашний помощник для проверки упражнений по русскому языку (Красная Ручка)

[![CI](https://github.com/4vertak/redpen-checker/actions/workflows/ci.yml/badge.svg)](https://github.com/4vertak/redpen-checker/actions/workflows/ci.yml)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**RedPen** — это эксперимент (гипотеза), который проверяет, может ли связка облачных AI‑сервисов помочь родителям учеников начальной школы быстро проверять домашние задания по русскому языку и объяснять ошибки простым языком. Он сравнивает ответ ребёнка с правильным ответом (эталоном) и выдаёт готовые объяснения ошибок. Если эталона нет, система просто найдёт ошибки и объяснит правила.

Прямо сейчас продукта ещё нет. Есть работающий фундамент, открытый код и желание проверить, насколько точно Yandex Vision распознает детский почерк, а DeepSeek — находит ошибки и даёт понятные пояснения.

## Для кого

- **Родители учеников 1-4 классов**, которые хотят помочь ребёнку с домашним заданием по русскому языку, но не помнят всех правил или не знают, как их объяснить.

Подробнее о пользователях — [документ персон](./docs/target-audience-and-personas.md).

## Что должно получиться в MVP

1. Родитель регистрируется в сервисе (email + пароль).
2. Входит в свой аккаунт.
3. Выбирает режим: **«Объяснить задание»** или **«Проверить готовое»**.

**Режим «Объяснить задание»:**
1. Родитель фотографирует страницу учебника или тетради с формулировкой упражнения.
2. Система распознаёт текст и выдаёт краткое, понятное объяснение, как его выполнять.

**Режим «Проверить готовое»:**
1. Родитель фотографирует страницу тетради с выполненным упражнением.
2. При желании указывает правильный ответ (эталон) или кратко описывает задание.
3. Система сравнивает ответ с эталоном (если он есть) или находит ошибки без эталона.
4. Родитель видит список ошибок с готовыми объяснениями и правилами.

**Ограничения MVP:**
- Работает только с **текстовыми заданиями** (диктанты, списывания, упражнения с пропущенными буквами).
- **Не поддерживает** графические разборы (подчёркивания членов предложения, морфемные схемы, рисунки).
- **Нет выбора учебника.** Система опирается на общие правила русского языка и контекст.
- **Обработка занимает 5–15 секунд.** В интерфейсе отображается индикатор прогресса.

## Архитектура

Веб-приложение на **Go + Gin** работает как оркестратор: получает фото -> отправляет в Yandex Vision API -> распознанный текст -> отправляет в DeepSeek API -> возвращает родителю результат с ошибками и пояснениями. Всё взаимодействие в MVP - синхронное.

## Технологический стек

- **Язык:** Go 1.26+
- **Веб-фреймворк:** Gin
- **База данных:** PostgreSQL (драйвер pgx/v5)
- **Миграции:** golang-migrate
- **AI-сервисы:** Yandex Vision API (OCR), DeepSeek API (анализ текста)
- **Контейнеризация:** Docker, Docker Compose
- **CI/CD:** GitHub Actions (линтер + тесты)

## Быстрый старт (локально)

### Предварительные требования
- Go 1.26+
- Docker и Docker Compose
- Ключи API:
  - [Yandex Vision API](https://cloud.yandex.ru/services/vision)
  - [DeepSeek API](https://platform.deepseek.com/)

### 1. Клонирование
```bash
git clone https://github.com/4vertak/redpen-checker.git
cd redpen-checker
```

### 2. Переменные окружения
Создайте файл `.env` в корне проекта:
```
PORT=8080
DATABASE_URL=postgres://postgres:postgres@localhost:5432/redpen?sslmode=disable
YANDEX_VISION_API_KEY=ваш_ключ
YANDEX_FOLDER_ID=ваш_папка_id
DEEPSEEK_API_KEY=ваш_ключ
JWT_SECRET=ваш_секретный_ключ
```

### 3. Запуск через Docker Compose
```bash
docker compose up -d
```
Проверьте: [http://localhost:8080/health](http://localhost:8080/health)

### 4. Ручной запуск (для разработки)
```bash
go run ./cmd/server/main.go
```
