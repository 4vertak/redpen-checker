# Use Case: Проверка одной работы

## Название
Проверка одной работы (Single Check)

## Участники
- Родитель (основной)
- Система «Red Pen»
- Внешние сервисы: OCR API (Yandex Vision), LLM API (DeepSeek)

## Предусловия
- Родитель авторизован в системе.
- Родитель находится на странице загрузки работы.

## Основной поток
1. Родитель выбирает режим: «Объяснить задание» или «Проверить готовое».
2. Родитель загружает фото тетради или учебника (JPG/PNG).
3. При желании указывает эталонный ответ или кратко описывает задание.
4. Система отправляет изображение в OCR API.
5. OCR API возвращает распознанный текст.
6. Система отправляет распознанный текст, задание и эталон (если есть) в LLM API.
7. LLM API возвращает результат: объяснение или список ошибок с пояснениями.
8. Система отображает результат.

## Альтернативные потоки
- **Ошибка OCR**: OCR API не смог распознать текст (статус "failed"). Система показывает сообщение «Не удалось распознать текст» и предлагает загрузить фото повторно.
- **Ошибка LLM**: LLM API вернул ошибку. Система показывает сообщение «Сервис проверки временно недоступен» и предлагает повторить попытку позже.».
- **Нестабильное соединение при загрузке**: Если во время отправки фото соединение прерывается, система сохраняет введённые данные формы и предлагает повторить загрузку.

## Постусловия
- Результат проверки сохранён в истории и доступен родителю.
