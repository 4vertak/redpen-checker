# Use Case: Проверка одной работы

## Название
Проверка одной работы (Single Check)

## Участники
- Учитель (основной)
- Система «Red Pen»
- Внешние сервисы: OCR API (GigaChat), LLM API (DeepSeek)

## Предусловия
- Учитель авторизован в системе.
- Учитель находится на странице загрузки новой работы.

## Основной поток
1. Учитель указывает предмет, текст задания и эталонный ответ (опционально).
2. Учитель загружает фото тетради (JPG/PNG).
3. Система отправляет изображение в OCR API.
4. OCR API возвращает распознанный текст.
5. Система отправляет распознанный текст, задание и эталон (если есть) в LLM API.
6. LLM API возвращает результат анализа: список ошибок с пояснениями.
7. Система отображает результат:
   - распознанный текст с подчёркнутыми ошибками,
   - список ошибок с пояснениями,
   - автоматически предложенную оценку.
8. Учитель может выполнить корректировку (отдельный запрос):
   - скорректировать распознанный текст,
   - добавить или удалить ошибки (опционально),
   - изменить предложенную оценку.
   Система сохраняет черновик изменений и ревизию, но не фиксирует ошибки до явного сохранения.
9. Учитель нажимает «Сохранить результат» (финальный запрос).
10. Система сохраняет работу со статусом "checked", фиксирует окончательный список ошибок в WorkError и отображает страницу истории проверок.
11. Учитель видит обновлённый список ранее проверенных работ.

## Альтернативные потоки
- **Ошибка OCR**: OCR API не смог распознать текст (статус "failed"). Система показывает сообщение «Не удалось распознать текст» и предлагает загрузить фото повторно.
- **Ошибка LLM**: LLM API вернул ошибку. Система показывает сообщение «Сервис проверки временно недоступен» и сохраняет работу со статусом "recognized" (распознанный текст доступен, проверка не выполнена). Учитель может запросить повторную проверку позже.
- **Нестабильное соединение при загрузке**: Если во время отправки фото соединение прерывается, система сохраняет введённые данные формы и предлагает повторить загрузку.

## Постусловия
- Проверенная работа сохранена в БД.
- Результат доступен в истории проверок.
